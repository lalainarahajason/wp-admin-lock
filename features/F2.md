# F2 — Masquage de la version WordPress
**Priorité : HAUTE**

## Objectif
Supprimer toute référence à la version WordPress : balise `<meta generator>`, query string `?ver=` sur les scripts/styles, en-tête `X-Powered-By`, flux RSS, page de login et tableau de bord de wordpress.
Empêcher les scanners (WPScan, etc.) de détecter la version exacte de WP.

## Mécanisme de "Hash Version"
Au lieu de supprimer `?ver=`, ce qui casse le cache navigateur, le plugin remplace la version par un hash MD5 court unique à l'instance (basé sur la `AUTH_KEY`).
*Exemple : `style.css?ver=6.4.1` devient `style.css?ver=a9f812`.*

## Actions
- `remove_action('wp_head', 'wp_generator');`
- Filtrer `script_loader_src` et `style_loader_src`.
- Supprimer la version dans les flux RSS (`the_generator`).
- Supprimer l'en-tête HTTP `X-Powered-By`.

## Classe : `LBS_HideVersion`
Fichier : `plugin/includes/Features/HideVersion.php`

## Points à supprimer

| Vecteur                        | Hook / filtre WordPress              |
|-------------------------------|--------------------------------------|
| `<meta name="generator">`     | `remove_action('wp_head', 'wp_generator')` |
| `?ver=` sur scripts/styles    | `style_loader_src` + `script_loader_src` filters |
| Flux RSS `<generator>`        | `the_generator` filter → retourner `''` |
| En-tête HTTP `X-Powered-By`   | `send_headers` action ou `header_remove()` |
| Page de login (générateur)    | `login_head` — supprimer wp_generator |

## Config JSON

```json
"hide_version": { "enabled": true }
```

## Contraintes
- Ne pas supprimer `?ver=` pour les assets des plugins si cela casse leur cache-busting (option : remplacer par un hash neutre)

## Critères d'acceptation
- [ ] La version est invisible dans le code source HTML.
- [ ] Aucune version WP dans le `<head>` front-end
- [ ] Paramètre `?ver=` retiré des URLs de scripts et feuilles de style front
- [ ] En-tête HTTP `X-Powered-By` supprimé
- [ ] Flux RSS ne contient plus la version
- [ ] Page de login ne révèle pas la version
- [ ] Aucune version WP dans le Dashboard admin
- [ ] Les assets (CSS/JS) conservent un paramètre de version (obfusqué) pour le cache.
