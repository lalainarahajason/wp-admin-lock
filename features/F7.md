### F7 — Blocage de l'énumération des utilisateurs
**Priorité : HAUTE**

## Objectif
Empêcher la découverte des logins WordPress via `/?author=N` et via l'endpoint API REST `/wp-json/wp/v2/users`. Ces deux vecteurs sont utilisés comme première étape d'une attaque brute force.

## Actions
1. Intercepter les requêtes `?author=n` et rediriger (301) vers la page d'accueil.
2. Désactiver le endpoint REST `/wp/v2/users` pour les requêtes non-authentifiées.
3. Supprimer les liens `rel="rest-alternate"` dans le `<head>`.

**Critères d'acceptation**
- [ ] Taper `monsite.com/?author=1` ne révèle pas le login dans l'URL ou sur la page.
- [ ] Requête `/?author=N` redirige vers la page d'accueil (sans révéler le login)
- [ ] L'endpoint `/wp-json/wp/v2/users` retourne HTTP 401 pour les non-connectés
- [ ] Les auteurs restent visibles sur les articles côté front si voulu (configurable)
- [ ] Un utilisateur connecté avec les droits adéquats accède toujours à la liste des users en admin
